## Anti-Patterns Verdict

**LLM assessment:** Mostly no AI slop on first glance — Plasma WebGL aurora, staggered rotated app panels, Fira
Sans/Fira Code pairing, Night Canvas palette all show deliberate craft. The tell is structural: `.landing-kicker` with
`text-transform: uppercase; letter-spacing: 0.12em` in Thesis Green appears on 6 sections (Features, Privacy, Roadmap,
Support, Universities, Install sub-header). The uniform section grammar (green kicker → h2 → muted copy → content) is
the AI grammar tell.

**Deterministic scan:** Exit 0, no findings. Absolute bans (gradient text, border-left stripe, glassmorphism) clean.

## Overall Impression

Real craft underneath a structural pattern that undercuts it. Rethinking section grammar and fixing mobile navigation
are the highest-leverage moves.

## What's Working

1. App showcase composition: staggered rotated panels with `appPanelIn` staggered entrance — strongest visual moment.
2. Night Canvas palette: cool-dark base with Thesis Green rationed correctly; progress bar gradient is semantically
   meaningful.
3. Reduced motion and accessibility plumbing: comprehensive `prefers-reduced-motion` block, consistent focus rings,
   `aria-labelledby` on support section.

## Priority Issues

**[P1] Kicker on every section**

- `.landing-kicker` (uppercase + tracked + green) opens Features, Privacy, Roadmap, Support, Universities, Install — 6
  instances. Absolute ban violation: AI grammar tell.
- Fix: Keep kicker in 2 sections max. Open others directly with h2, a version number, an icon, or a structural
  alternative.
- Command: /impeccable layout

**[P1] Mobile navigation collapses to nothing**

- At 760px, nav-links and nav-cta both `display:none`. Logo + language toggle only. No hamburger, no anchor strip.
- Fix: Minimal hamburger or bottom anchor strip on mobile.
- Command: /impeccable adapt

**[P2] Muted text contrast failures**

- `.landing-hero-social` at `rgba(221,221,240,0.4)` ≈ 3.6:1 against #0d0d11 — fails WCAG AA. `.landing-section-copy` at
  0.55 is borderline.
- Fix: Raise social proof text to minimum 0.58 opacity. Audit section copy with gradients accounted for.
- Command: /impeccable audit

**[P2] Scroll-driven reveal animations gate content in Chrome**

- `.landing-reveal` starts at `opacity:0` inside `@supports (animation-timeline:view())`. Background tabs, headless
  renderers, and social preview scrapers may see blank sections.
- Fix: Remove opacity:0 initial state or add JS intersection observer fallback.
- Command: /impeccable harden

**[P2] Section flow buries install behind donation ask**

- Flow: Hero → Features → Privacy → Roadmap → Support (donate) → Universities/Install. Support comes before the "where
  it works and how to install" section.
- Fix: Move Universities/Install before Support.
- Command: /impeccable layout

## Minor Observations

- Nav `::before` scroll-progress bar is a nice touch but the full-opacity Thesis Green stroke feels heavier than system
  norms.
- Third hero CTA "See what's changed" reads as changelog, not feature discovery — relabel to "See how it works".
- Social proof claim is generic without a specific number — consider "310 active installs" over implied avatars.
